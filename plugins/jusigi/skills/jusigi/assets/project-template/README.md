# Jusigi automation

This repository was generated by the Jusigi Skill as a safe baseline for Korean stock research and paper-trading automation on GitHub Actions.

## Current state

- Broker selection: `__JUSIGI_BROKER__`
- Live trading: **disabled**
- Short selling: **disabled**
- Core/satellite allocation: `__JUSIGI_CORE_RATIO__% / __JUSIGI_SATELLITE_RATIO__%`
- Deterministic risk gate and dry-run broker included
- Scheduled report and paper-trading workflows included

The starter deliberately does not contain or generate a live order endpoint or an investment recommendation. Add current market/news and paper-broker adapters only from official provider documentation, then test them in a paper environment.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
PYTHONPATH=src python -m unittest discover -s tests -v
PYTHONPATH=src python -m jusigi.main healthcheck
```

Edit `config/investment-policy.yaml` to add an approved ticker allowlist and non-secret limits. Do not store credentials in that file.

Optional notifications use these GitHub Secrets:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

Paper-provider and OpenAI secret names must be added only after their adapters are implemented from current official documentation. Configure values through repository Settings or `gh secret set`; never paste them into source, issues, logs, or chat.

## Workflows

- `ci.yml`: network-free tests
- `market-report.yml`: weekday status/report entry point
- `paper-trade.yml`: guarded paper decision entry point

GitHub schedules are best-effort and may be delayed. Keep external monitoring and reconciliation for any time-sensitive workflow.

## Disclaimer

This is automation software, not investment advice or a promise of returns. Validate data licenses, broker eligibility, taxes, account rules, risk limits, and operational behavior independently before considering live use.
